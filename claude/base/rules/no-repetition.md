# No repetition — single source, link elsewhere

Every concept or piece of information belongs in exactly one section. When that content is relevant elsewhere, link to it — don't copy it.

## Rules

- **One canonical section per topic.** If a concept needs explanation, explain it once in the most appropriate section. Every other mention is a link.
- **No summary rehash.** Don't restate in a summary what was already said in the body. If a summary section exists, it should contain links or one-line pointers, not paraphrased duplicates.
- **No intro repetition.** Don't repeat in the introduction what the body will cover in full. A one-sentence framing is fine; a paragraph that mirrors the body is not.
- **Cross-references over copy-paste.** When a later section depends on context from an earlier one, write "see [Section Name](#anchor)" rather than re-explaining it inline.

## Applies to

Design docs, RFCs, runbooks, incident write-ups, and any multi-section technical document.
